# 区域科创教育服务平台需求文档

## 一、项目背景与定位

### 1.1 项目背景

随着国家对科技创新人才培养的重视，教育部白名单竞赛成为衡量学生科创能力的重要维度。当前区域内的科创教育存在以下痛点：

• **家长端**：学生科创成果分散，缺乏系统化成长记录和可视化画像

•
**学校/区域端**：无法掌握区域内学生科创整体水平，缺乏数据支撑的教育决策

• **资源端**：赛事信息分散，报名入口不一，缺乏统一入口

• **学习端**：学生缺乏与赛事对接的系统性课程和实践平台

### 1.2 平台定位

**区域级科创教育综合服务平台**，构建\"数据-资源-学习-实践\"的完整生态闭环，服务对象包括：

• **学生/家长**：成长记录、课程学习、赛事实践

• **学校**：学生管理、教学资源、数据分析

• **区域教育局**：整体监控、政策落地、质量评估

## 三、功能模块详解

### 3.1 家长端小程序模块

**核心目标**：降低家长使用门槛，实现学生科创成果的系统化沉淀

#### 3.1.1 用户认证与绑定

• 手机号注册/登录

• 学生信息录入（姓名、学校、年级、学籍号）

• 家长与学生账号关联（支持多子女绑定）

#### 3.1.2 资料上传功能

**上传类型**：

• 科创类获奖证书（图片/PDF）

• 学科成绩单（支持批量导入）

• 项目作品（代码文件、作品照片、演示视频）

• 参与活动记录

**上传流程**：

1\. 选择资料类型

2\. 拍照/上传文件

3\. 自动识别（OCR提取证书关键信息：奖项名称、时间、级别）

4\. 手动校验确认

5\. 提交入库

#### 3.1.3 成长档案查看

• **时间轴展示**：按时间线展示学生科创成长历程

• **数据统计**：获奖数量、参赛次数、课程学习时长

• **能力雷达图**：基于成果数据生成多维度能力画像

> ◦ 创新思维
>
> ◦ 编程能力
>
> ◦ 团队协作
>
> ◦ 项目实践
>
> ◦ 表达展示

#### 3.1.4 消息通知

• 赛事报名提醒

• 课程更新通知

• 成果审核结果

• 学习进度报告

• 赛事报名结果提醒

#### 3.1.5 赛事报名

• 赛事报名缴费

• 查看赛事详情

### 3.2 平台管理中心模块

**核心目标**：提供权威赛事信息和便捷报名通道

#### 3.2.1 教育部白名单赛事库

• **赛事分类**：

> ◦ 编程类：NOI、CSP-J/S、蓝桥杯等
>
> ◦ 机器人类：WER、RoboRAVE等
>
> ◦ 创客类：全国青少年科技创新大赛等
>
> ◦ 人工智能类：AI创新挑战赛等

• **赛事详情页**：

> ◦ 赛事简介与背景
>
> ◦ 主办方与官方链接
>
> ◦ 参赛对象与组别
>
> ◦ 竞赛时间轴
>
> ◦ 历届获奖案例
>
> ◦ 备赛指南

#### 3.2.2 报名管理系统

• **报名流程**：

1\. 学生/家长选择赛事

2\. 阅读赛事规则与要求

3\. 填写报名信息（可自动同步成长档案中的基础信息）

4\. 上传报名所需材料

5\. 提交审核

6\. 支付报名费（如有）

7\. 获取报名凭证

• **报名状态追踪**：

> ◦ 待审核 → 审核中 → 审核通过 → 报名成功 → 参赛中 → 结果公示

#### 3.2.3 赛事服务

• 赛事资讯推送

• 备赛资源推荐

• 在线模拟测试

• 历年真题库

### 3.3 在线编程实践平台

**核心目标**：提供与赛事对接的编程环境，支持三种主流语言的学习与实践

#### 3.3.1 图形化编程环境（Scratch/Blockly）

• **适用对象**：小学低年级、零基础学生

• **功能特性**：

> ◦ 拖拽式积木编程界面
>
> ◦ 实时代码预览
>
> ◦ 内置项目案例库
>
> ◦ 作品分享社区
>
> ◦ 支持导出为可执行文件

#### 3.3.2 Python编程环境

• **适用对象**：小学高年级、初中生

• **功能特性**：

> ◦ 在线代码编辑器（支持语法高亮）
>
> ◦ 交互式运行环境（Jupyter Notebook模式）
>
> ◦ 内置常用库（turtle、pygame、numpy等）
>
> ◦ 代码自动补全
>
> ◦ 错误提示与调试辅助

#### 3.3.3 C++编程环境

• **适用对象**：初中生、高中生（竞赛备考）

• **功能特性**：

> ◦ 在线代码编辑器
>
> ◦ 多版本编译器支持（C++11/14/17）
>
> ◦ 标准输入输出模拟
>
> ◦ 在线评测系统（OJ）
>
> ◦ 性能分析与优化建议

#### 3.3.4 统一功能特性

• 题目分类体系（基础题、进阶题、竞赛真题）

• 在线提交与自动评测

• 代码保存与版本管理

• 学习路径推荐

• 排行榜与成就系统

### 3.4 课程学习中心

**核心目标**：提供系统性课程体系，支持自主学习路径

#### 3.4.1 课程分类

• 笔记与收藏

• 答疑社区

• 学习报告生成

• AI学习助手（智能答疑、错题分析）

## 四、数据模型与业务流程

### 4.1 核心数据实体

学生档案\
├── 基础信息（姓名、学校、年级）\
├── 学科成绩（时间、科目、分数）\
├── 获奖记录（赛事名称、奖项级别、获奖时间、证书）\
├── 学习数据（课程学习时长、练习完成情况、代码提交次数）\
├── 项目作品（作品名称、类型、作品链接、创作时间）\
└── 能力画像（创新思维、编程能力、团队协作等维度评分）\
\
课程体系\
├── 课程分类（语言类型、难度级别）\
├── 课程内容（视频、图文、练习）\
├── 学习路径（前置课程、后续课程）\
└── 学习记录（学生-课程关联）\
\
赛事库\
├── 赛事信息（名称、主办方、时间、规则）\
├── 报名记录（学生-赛事关联、状态）\
├── 备赛资源（真题、教程）\
└── 成绩记录（获奖信息）

### 4.2 核心业务流程

**成长档案构建流程**：

家长上传资料 → OCR识别信息 → 人工/自动审核 → 数据入库\
→ 能力模型计算 → 生成成长档案 → 持续更新迭代

**赛事报名流程**：

赛事发布 → 家长/学生浏览 → 选择赛事 → 填写报名信息\
→ 提交审核 → 审核通过 → 报名成功 → 参赛 → 结果录入

**学习与实践流程**：

能力测评 → 推荐学习路径 → 观看课程 → 完成练习 → 编程实践\
→ 项目实战 → 能力验证 → 更新学习路径

## 五、平台特色与创新点

### 5.1 AI驱动的个性化推荐

• 基于学生成长数据，智能推荐适配的课程和赛事

• 学习路径动态调整，根据掌握程度推荐进阶内容

### 5.2 科创能力可视化画像

• 多维度能力雷达图

• 同学段/同区域对比分析

• 成长轨迹可视化

### 5.3 区域数据看板

• 区域学生科创整体水平分析

• 学校科创实力排名

• 赛事参与度统计

• 政策效果评估

### 5.4 闭环成长体系

• 学习 → 实践 → 参赛 → 获奖 → 记录 → 再学习的完整闭环

• 成果沉淀反哺学习路径优化

## 六、非功能性需求

### 6.1 性能要求

• 支持万级并发用户在线编程

• 页面加载时间 \< 2秒

• 代码编译响应时间 \< 5秒

### 6.2 安全要求

• 学生数据隐私保护

• HTTPS加密传输

• 代码防抄袭机制

• 权限分级管理

### 6.3 兼容性要求

• 家长小程序支持iOS/Android

• 学生端支持主流浏览器（Chrome、Firefox、Edge、Safari）

• 编程环境兼容移动端和PC端

  ----------------------------------- --------------------------------------------------
